# Internal Operating Procedure

This procedure is intended to support an evidence-based attribution position. It must be followed consistently before high attribution percentages can be defended.

For each relevant SAP IS-U ticket, project task or technical request:

1. Open SAP IS-U Assistant.
2. Select the active client.
3. Create or update an incident/work item with an anonymised ticket reference.
4. Classify SAP module and process.
5. Record affected SAP objects and IDs where appropriate.
6. Search similar incidents.
7. Use AI/RAG technical query if needed.
8. Use combined Standard plus permitted client knowledge where appropriate.
9. Review generated output manually.
10. Validate the conclusion in the SAP system manually.
11. Prepare Jira, email, debug checklist or documentation output.
12. Mark whether the output was used in client delivery.
13. Record actual time spent.
14. Estimate time without the tool.
15. Record estimated time saved.
16. Rate usefulness and, when possible, accuracy.
17. Store usage ID and evidence link.
18. Generate monthly IP usage report.

The tool should be treated as the central internal workflow for software-enabled SAP IS-U technical analysis. Work not processed through the tool should be treated conservatively or excluded from high attribution claims.
