# SAP IS-U Assistant IP Box Position Paper

## Purpose

This document is an internal position paper for qualified Cyprus tax advisors. It does not determine eligibility, qualifying profits, nexus fraction, transfer pricing, tax savings or filing treatment. It describes why the SAP IS-U Assistant may be relevant to a Cyprus IP Box analysis if the factual evidence supports it.

Reference baseline: the Cyprus IP Box regime may cover copyrighted software and may include IP income embedded in services or processes directly related to qualifying IP assets. Advisor validation is required before any tax position is taken.

## Asset Overview

SAP IS-U Assistant is a proprietary software tool developed for internal SAP IS-U consulting delivery. It combines a local web UI, client-aware knowledge storage, Qdrant vector search, OpenAI-assisted synthesis, incident registration, evidence management, research-agent ingestion, Kanban operations and finance support.

The current exploitation model is internal use by the company in SAP IS-U consulting work. It is not currently sold as SaaS or licensed externally. Future commercialization options may include private SaaS, desktop licensing, managed support services or partnerships with SAP consultancies.

## Why It May Be Copyrighted Software

The qualifying asset for review is the original software layer developed by the company: source code, architecture, orchestration logic, prompts, UI flows, storage model, incident workflows, research-agent pipeline, client isolation, standard/client knowledge separation, dossier generation and tests.

The company does not own SAP software, SAP documentation, SAP transactions, SAP tables, OpenAI models, Qdrant, third-party libraries or client confidential data.

## Service Delivery Role

The tool is intended to become the central internal workflow for software-enabled SAP IS-U technical analysis. The strongest position requires evidence that relevant SAP IS-U outputs are generated, documented or materially assisted through the tool.

Current implemented capabilities include:

- Standard and client-scoped knowledge bases.
- RAG-based chat with source retrieval.
- Ingesta and review workflow for KB approval and Qdrant indexing.
- SAP IS-U incident registration with evidence and annual IP Box dossier PDF.
- Research/crawler agents that populate Standard KB with SAP IS-U reference knowledge.
- Kanban ticket workflow.
- Finance module for operational tracking.
- Initial IP Box usage logging/reporting modules.

Planned or partially implemented evidence controls include deeper UI integration for usage logs, manual output-used confirmation, accuracy scoring, ticket-level evidence exports and monthly advisor packs.

## Income Streams for Advisor Review

Potentially included:

- SAP IS-U technical analysis materially assisted by the tool.
- SAP IS-U incident resolution work processed through incident/KB/chat workflows.
- Client delivery outputs generated, drafted, checked or documented by the tool.
- SAP IS-U knowledge engineering that improves the proprietary software asset.

Excluded:

- Pure administration.
- Sales and negotiation.
- Invoicing and finance administration.
- Non-technical meetings.
- Manual SAP consulting not materially assisted by the tool.
- Work unrelated to SAP IS-U or the proprietary software asset.
- Any income derived from SAP IP, client data or third-party tools rather than the company software layer.

## Maximum Position - Up To 100%

This is an aggressive advisor-assessed position. It is supportable only if almost all relevant SAP IS-U service delivery is processed through SAP IS-U Assistant and the tool is demonstrably central to the output. Required evidence includes usage logs, ticket-level evidence, client work references, output-used confirmation, monthly reports and revenue mapping.

## Target Position - 60%

This is the internal target position. It requires strong but not absolute evidence that a substantial majority of productive SAP IS-U work is assisted by the tool. A plausible model is:

IP Attribution % = assisted productive hours / total productive SAP IS-U hours x software contribution factor x qualifying service factor.

The target should be validated through several months of real logs and monthly reports.

## Conservative Fallback - 25%-40%

This may be appropriate if advisors view the tool as a meaningful internal software asset but not sufficiently central to justify the target or maximum positions. The fallback should still be evidence-based and should exclude non-assisted, administrative or non-technical work.

## Advisor Validation Required

The final assessment must cover legal ownership, qualifying intangible asset status, qualifying profit, nexus fraction, service-line attribution, transfer pricing, documentation standards and Cyprus tax treatment.
