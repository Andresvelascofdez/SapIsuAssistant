# Project Instructions for AI Coding Agents

- Preserve existing functionality.
- Keep changes modular and narrowly scoped.
- Do not remove or simplify SAP IS-U Assistant features.
- Do not hardcode real client data, API keys, tickets, invoices or confidential examples.
- Do not invent evidence, dates, logs, hours, commits, screenshots or usage.
- Use anonymised placeholders such as CLIENT_A, CLIENT_B, TCKXXXXX and HNDXXXXX.
- IP Box documentation must be written in professional English.
- IP Box documents may describe target positions, but final eligibility, qualifying profit, nexus fraction and attribution must be left for qualified Cyprus tax advisors.
- Code should avoid unnecessary comments.
- Tests should be run where possible.
- If uncertain, create a TODO/TBC section rather than making unsupported claims.
- Keep SAP standard/public knowledge separate from client-specific Z/private knowledge.
- Research/crawler output belongs in Standard KB; client-specific content should be ingested manually into the relevant client scope.
